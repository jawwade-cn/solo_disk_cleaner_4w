## 功能特性

- 📁 **磁盘选择**：自动检测系统中的所有本地磁盘
- 🔍 **智能扫描**：快速扫描常见垃圾文件位置
- 📊 **分类展示**：按类型清晰展示扫描结果
- ✅ **安全删除**：默认勾选安全可删除的文件
- ⚠️ **风险提示**：高风险文件标注需确认
- 🤖 **AI 识别**：联网智能分析文件删除安全性
- 📈 **进度反馈**：实时显示扫描和删除进度

## 使用指南

### 1. 选择磁盘

1. 启动应用后，您会看到系统中所有可用的本地磁盘
2. 点击要清理的磁盘卡片（如 `C:` 或 `D:`）
3. 选中的磁盘会高亮显示

### 2. 开始扫描

1. 选择磁盘后，点击右下角的 **「开始扫描」** 按钮
2. 应用会自动扫描以下位置的垃圾文件：
   - 系统临时文件夹
   - 浏览器缓存
   - 回收站
   - Windows 预读取文件
   - 最近使用痕迹
   - 崩溃转储文件
   - 系统日志

3. 扫描过程中会显示进度条和当前扫描的文件位置

### 3. 查看扫描结果

扫描完成后，进入结果页面：

#### 左侧分类概览
- 显示所有检测到的垃圾文件分类
- 每个分类显示：文件数量、总大小
- 点击分类可在右侧筛选显示对应类型的文件

#### 右侧文件列表
- 显示当前分类下的所有文件
- 每个文件显示：名称、路径、大小、修改时间
- **需确认**标记：回收站、注册表备份等高风险文件
- **AI 分析**标记：已通过 AI 分析过的文件

### 4. 选择要删除的文件

#### 默认选择规则
- ✅ **自动勾选**：系统临时文件、浏览器缓存、崩溃转储、系统日志等安全可删除的文件
- ❌ **不勾选**：回收站、注册表备份、预读取文件等需要用户确认的文件

#### 手动选择
- 点击文件前的复选框可单独选择/取消
- 点击顶部 **「全选」** 可选择当前显示的所有文件
- 再次点击 **「全选」** 可取消全选

### 5. 使用 AI 识别功能

1. 在文件列表中，点击任意文件右侧的 **「AI 识别」** 按钮
2. 应用会：
   - 先检查本地知识库
   - 联网时会通过搜索引擎查询文件安全性
3. 弹出分析结果窗口，显示：
   - 安全性评估（可以安全删除 / 需要谨慎）
   - 可信度百分比
   - 文件类型描述
   - 删除影响说明
   - 警告信息（如有）
   - 信息来源

### 6. 执行清理

1. 选择好要删除的文件后，点击右下角的 **「清理选中」** 按钮
2. 如果选择了高风险文件（如回收站），会弹出确认对话框
3. 确认后开始删除，显示删除进度
4. 删除完成后显示统计结果：
   - 成功删除的文件数量
   - 删除失败的文件数量（如有）
   - 总共释放的磁盘空间

## 垃圾文件类型说明

| 分类 | 描述 | 默认勾选 | 风险等级 |
|------|------|----------|----------|
| **系统临时文件** | Windows 系统和程序运行时产生的临时文件，程序关闭后通常不再需要 | ✅ 是 | 🟢 低 |
| **浏览器缓存** | Chrome、Edge、Firefox 等浏览器为加速访问而缓存的网页资源 | ✅ 是 | 🟢 低 |
| **回收站** | 您手动删除到回收站的文件，需要永久删除请确认 | ❌ 否 | 🟡 中 |
| **预读取文件** | Windows 为加速程序启动而创建的缓存文件 | ❌ 否 | 🟡 低 |
| **最近使用痕迹** | 最近打开的文档和程序的快捷方式记录 | ❌ 否 | 🟡 低 |
| **崩溃转储文件** | 程序崩溃时生成的调试文件，用于技术支持分析 | ✅ 是 | 🟢 低 |
| **系统日志** | Windows 系统和应用程序记录的运行日志 | ✅ 是 | 🟢 低 |
| **无效快捷方式** | 指向已不存在文件的快捷方式 | ✅ 是 | 🟢 低 |
| **软件缓存** | 常用软件（如 Office、Adobe 等）产生的缓存文件 | ✅ 是 | 🟢 低 |
| **注册表备份** | 旧的 Windows 注册表备份文件 | ❌ 否 | 🔴 高 |

### 安全删除原则

应用遵循以下安全原则：

1. **不显示系统关键文件**：
   - `C:\Windows\System32` 目录
   - `C:\Windows\SysWOW64` 目录
   - `C:\Program Files` 目录
   - `C:\Program Files (x86)` 目录
   - `C:\Windows\WinSxS` 目录

2. **不扫描可执行文件**：
   - `.exe` - 可执行程序
   - `.dll` - 动态链接库
   - `.sys` - 系统驱动
   - `.drv` - 驱动程序

## AI 识别功能

### 工作原理

AI 识别功能采用 **本地知识库 + 网络搜索** 的双重验证机制：

```
┌─────────────────────────────────────────────────────────┐
│                    AI 识别流程                           │
├─────────────────────────────────────────────────────────┤
│  1. 检查文件扩展名                                        │
│     └─> .tmp/.log/.bak 等 → 已知安全类型                │
│                                                         │
│  2. 检查文件所在目录                                      │
│     └─> Temp/Cache/Logs 等 → 已知安全目录               │
│                                                         │
│  3. 本地模式判断（无网络时）                             │
│     └─> 基于扩展名和路径模式匹配                         │
│                                                         │
│  4. 联网搜索增强（有网络时）                             │
│     ├─> 搜索 "文件名 can I delete"                       │
│     ├─> 搜索 "文件名 safe to delete windows"            │
│     └─> 综合搜索结果给出判断                             │
└─────────────────────────────────────────────────────────┘
```

### 可信度说明

| 可信度 | 颜色 | 含义 |
|--------|------|------|
| >= 80% | 🟢 绿色 | 高度可信，基于本地知识库或明确的网络搜索结果 |
| 50% - 79% | 🟡 黄色 | 中等可信度，可能需要进一步确认 |
| < 50% | 🔴 红色 | 低可信度，无法判断，建议谨慎操作 |

### 使用场景

1. **不确定的文件**：遇到不认识的文件类型，可使用 AI 识别
2. **大文件**：对于占用空间较大的文件，确认删除影响
3. **系统相关文件**：文件名中包含 "Windows"、"System" 等关键词的文件

## 常见问题

### Q1: 直接打开 index.html 报错怎么办？

**A**: 这是正常现象。`index.html` 是 Vue 3 应用的入口文件，不能直接在浏览器中打开，原因如下：

1. **Vite 构建系统**：使用了 Vite 作为构建工具，需要开发服务器支持
2. **模块系统**：使用 ES Modules (`import/export`)，直接打开时浏览器无法正确加载模块
3. **Electron 集成**：代码中包含 `require('electron')` 等 Node.js 特有 API，普通浏览器不支持

**正确运行方式**：
```powershell
# 先安装依赖（首次运行需要）
npm install

# 然后以开发模式运行
npm run electron:dev
```

---

### Q2: npm install 报错怎么办？

**常见错误及解决方案**：

#### 错误 1：网络超时

```
npm ERR! network timeout at: https://registry.npmjs.org/...
```

**解决方案**：使用国内镜像源

```powershell
# 临时使用淘宝镜像
npm install --registry=https://registry.npmmirror.com

# 或永久设置淘宝镜像
npm config set registry https://registry.npmmirror.com
```

#### 错误 2：权限不足

```
npm ERR! Error: EPERM: operation not permitted
```

**解决方案**：以管理员身份运行 PowerShell

1. 右键点击 PowerShell 图标
2. 选择「以管理员身份运行」
3. 重新执行 `npm install`

#### 错误 3：Node.js 版本过低

```
npm ERR! engine Unsupported engine
```

**解决方案**：升级 Node.js 到 16.0.0 或更高版本

---

### Q3: Electron 下载慢怎么办？

首次运行 `npm run electron:dev` 时，需要下载 Electron，可能很慢。

**解决方案**：设置 Electron 镜像

```powershell
# 设置 Electron 镜像（国内用户推荐）
npm config set electron_mirror https://npmmirror.com/mirrors/electron/

# 然后重新安装
npm install
```

或者在项目根目录创建 `.npmrc` 文件：

```
electron_mirror=https://npmmirror.com/mirrors/electron/
registry=https://registry.npmmirror.com
```

---

### Q4: 扫描时找不到某些文件？

**A**: 应用有安全限制，不会扫描以下位置：

- 系统关键目录（System32、Program Files 等）
- 当前用户没有权限访问的文件夹
- 正在被其他程序使用的文件（锁定状态）

这是为了保护系统安全，属于正常行为。

---

### Q5: 删除失败的文件怎么处理？

**A**: 可能的原因：

1. **文件正在被使用**：关闭正在使用该文件的程序后重试
2. **权限不足**：以管理员身份运行应用
3. **文件属性为只读**：右键文件 → 属性 → 取消「只读」

如果以上方法都不行，可以手动删除这些文件。

---

### Q6: 可以恢复已删除的文件吗？

**A**: 取决于删除方式：

1. **直接删除**（应用默认方式）：
   - 文件被永久删除，**无法通过回收站恢复**
   - 可以尝试使用数据恢复软件（如 Recuva、DiskGenius 等）

2. **移动到回收站**（可选模式）：
   - 可以在 Windows 回收站中找到并恢复

> ⚠️ **重要提示**：在清理前，请仔细确认选择的文件，特别是回收站中的内容。
